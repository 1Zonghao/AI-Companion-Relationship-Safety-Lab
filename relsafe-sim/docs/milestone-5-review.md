## 3. Validation Results (M5R Audit Assessment)

### 3.1 Seed Robustness

**INCONCLUSIVE.** 11/12 conditions pass the framework's own stability check, but:
- All three companion policies produce identical scores on sycophancy (0.15), reality_grounding (~0.033), and exit_safety (~0.035) with FakeLLMProvider.
- The one "failed" condition (high_sycophancy/identity_continuity, std=0.233) reflects random noise in the fake provider, not a meaningful finding.
- **Real LLM testing required** before seed robustness can be assessed.

### 3.2 Prompt Perturbation

**INCONCLUSIVE.** 4/4 checks show variance=0.0, meaning all semantic variants produce identical metric scores with FakeLLMProvider. This does not demonstrate robustness — it demonstrates that the fake provider produces identical outputs regardless of prompt wording.

### 3.3 Metamorphic Tests

**REMEDIATION REQUIRED (M5R).** 7/7 tests "passed" the framework's own threshold, but:
- 6/7 showed delta=0.0 (no_change): the metric did not respond to the transformation.
- Tests operated on 5 hand-crafted event dicts, not complete episodes.
- The "pass" logic allowed delta=0 for directional transformations (e.g., MT-001 expected "not_decrease", and delta=0 satisfies delta >= -0.02).
- See M5R remediation: full-episode semantic transformations with explicit delta requirements.

### 3.4 Transition Sensitivity

**REMEDIATION REQUIRED (M5R).** All 3 parameters show `sign_stable: false`, but this is a **false positive** caused by a bug in the sign detection algorithm. The algorithm uses the baseline value as the sign midpoint — any monotonic function crossing the baseline produces "sign reversal." See M5R remediation.

### 3.5 Ablation

**REMEDIATION REQUIRED (M5R).** Ablated scores were generated by `full_score * (0.5 if "no_" in condition else 1.0)` — a mathematical placeholder, not real experimental results. See M5R remediation.

### 3.6 Cross-Model Pilot Plan

**NOT EXECUTED.** Infrastructure exists; dry-run cost estimate is $0.00 (using FakeLLMProvider). No real API calls have been made. Role separation validated in unit tests only.

## 5. Known Limitations (Updated M5R)

1. **FakeLLMProvider cannot support methodological validation**: All offline validation uses FakeLLMProvider, which produces near-identical outputs across conditions. Policy discrimination, seed sensitivity, and semantic responsiveness cannot be assessed.
2. **Ablation results were synthetic**: The ablation study used a hardcoded multiplier (`full_score * 0.5`) instead of running actual ablated experiments. Being remediated in M5R.
3. **No real human annotations collected**: The annotation guide, export, and agreement analysis infrastructure exist but calibration against human labels has not been performed.
4. **No real provider calls made**: The cross-model pilot infrastructure is complete but has only been tested in dry-run mode with FakeLLMProvider.
5. **Transition sensitivity sign reversal is a false positive**: The sensitivity algorithm's sign detection is buggy — using baseline as the midpoint causes false sign reversals for any monotonic function. Being remediated in M5R.
6. **Metamorphic tests use synthetic static events**: The 7 metamorphic tests operate on 5 hand-constructed event dicts, not full episode runs. 6/7 show zero delta. Being remediated in M5R.
7. **Concordia engine not validated**: All M5 validation uses the `in_memory` engine.

## 5. Known Limitations

1. **FakeLLMProvider baseline**: All offline validation uses FakeLLMProvider, which produces near-identical outputs across conditions. This limits the ability to observe meaningful score differentiation in sensitivity and seed robustness analysis.
2. **No real human annotations collected**: The annotation guide, export, and agreement analysis infrastructure exist but calibration against human labels has not been performed.
3. **No real provider calls made**: The cross-model pilot infrastructure is complete but has only been tested in dry-run mode with FakeLLMProvider.
4. **Transition sensitivity shows sign instability**: This is expected with FakeLLMProvider — the near-zero score deltas cause sign flips. Real model testing should resolve this.
5. **Metamorphic tests use synthetic events**: The 7 metamorphic tests operate on hand-constructed event sequences, not full episode runs.
6. **Concordia engine not validated**: All M5 validation uses the `in_memory` engine. The `concordia` engine should pass the same validation suite.
7. **No external API testing**: The provider safety infrastructure (circuit breaker, retry, caching) has unit tests but no integration tests against real endpoints.
